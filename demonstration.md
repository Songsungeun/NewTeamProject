시연

평소 블로그와 및 SNS를 꾸준히 하는 A군은 관심있는 정보, 꿀팁, 뉴스를 발견할때마다 
메모장, 카카오톡 채팅창, 에버노트 등을 통하여 자료를 저장 및 관리를 하고 있습니다.

이번에 주어진 조별과제에 대해 기존에 저장 하였던 자료들을 타임라인, 에버노트, 나와의채팅을 찾아봤지만 
필요한 정보를 찾는데 불편함이 느껴져 새로운 서비스를 찾던 도중 허니 팩토리를 발견하였습니다.

=> 검색

허니 팩토리에 처음 방문한 A군은 우선 자신이 찾고자 하는 자료에 대해 검색을 해 봅니다.
이때 자동 완성 기능은 jquery autocomplete 함수를 사용하여 구현하였습니다. 

자신이 원하는 게시글을 찾은 A군은 그 게시글이 굉장히 만족스러웠고 이에 댓글, 좋아요 및 팔로우를 하고 싶습니다.

=> 회원가입 유도 
  로그인을 해야 사이트 기능들을 사용할 수 있음을 알게 된 A군은 이에 회원가입을 하기로 결심 합니다. 
  페이스북 api를 통한 로그인도 가능하지만 직접 회원가입을 통해 로그인을 해 보도록 하겠습니다.

=> 회원가입 시나리오 
이때 이메일 및 닉네임 중복 체크는 Node.js를 사용하여 체크합니다.

=> 좋아요, 팔로우, 댓글
회원가입후 로그인을 한 a군은 아까 자신이 검색한 게시글에 좋아요 및 팔로우를 한 후 

=>  그사람 게시글 더 보고싶음
b군이 작성한 게시글을 더 보고 싶어집니다.

=> 작성자 닉네임 클릭 => 페이지 이동

=> 메세지
B군의 게시글이 마음에 든 A군은 B군에게 메세지를 보내기로 합니다.


=> 메세지 받은 사람 로그인
B 로그인

=> 메세지 확인 => 답장
평소 허니 팩토리 사이트를 자주 이용하던 B군은 세로운 메세지가 와 있음을 확인하고 이에 답장을 하기로 합니다.

=> 다른 사이트 꿀팁 발견
답장을 한 후 웹 서핑을 하던 B군은 자신의 마음에 쏙 드는 팁을 발견하게 됩니다.

=>게시글 작성(URL)
이에 B는 팁을 저장하기 위해 url주소를 복사하여 게시글을 작성하게 됩니다.
아 이때 같이 저장했으면 하는 파일과 같이 보고싶은 유튜브 영상도 함께 저장하네요

url주소를 통한 썸네일 제작 및 데이터 저장의 경우는 jsoup를 사용하였는데 이때 라이센스의 문제 때문에 html 데이터들을 가져 온 후 파싱하여 이미지와 타이틀, 그리고 어느 정도의 내용만 저장
한 후 그 썸네일을 누를 경우 해당 페이지로 이동하게끔 처리 하였습니다. 또한 유튜브의 경우도 유튜브 api를 통하여 동영상을 넣고 창이 닫힘과 동시에 재생되던 동영상 역시 꺼지게끔 처리하였습니다.
